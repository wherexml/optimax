## 一、核心判断



- **定位：**我们要构建的是一个叠加在现有 ERP/SRM/自研底座之上的风险运营与决策层平台，面向大型运营商、制造业和港口等企业客户，打造以**高质量情报数据输入、供应链知识图谱为底座、以 AI Agent 为工作引擎、以运筹学优化为核心差异化能力**的"**风险预警与协同处置平台**"

- 基于我们对产品的理解，快速构建起第一个剧本完整能力的产品蓝图，同时把第一批客户需求做成我们的“灯塔项目 + 行业底座”。以外部风险 intelligence 切入，以内部规则联动落地，以 OR 优化能力形成高差异化，以 Agent 驱动运营效率，把项目做成平台、把平台做成方法论、把方法论再复制到运营商与复杂制造行业。

## 二、战略定位

**一句话定位：**

```
面向大型国央企复杂供应链场景，打造叠加在现有 ERP/SRM/电子采购平台之上的“风险 intelligence、业务影响量化与协同处置决策层平台”。
```

愿景不止是"看见风险"，而是四层递进：

| 层次 | 能力 | 说明 |
|------|------|------|
| 看见 | 持续感知外部与内部风险 | 采集、清洗、归类、去重 |
| 看懂 | 关联到供应商、物料、订单、产品、区域与责任人 | 实体映射、知识图谱 |
| 算清 | 量化影响程度、传播路径与处置代价 | **运筹学算子的核心价值区** |
| 闭环 | 把建议转成待办、审批、处置动作与复盘沉淀 | 闭环运营 |



## 三、差异化："运筹学 + AI Agent"的组合

竞品矩阵里最关键的割裂：**情报层强的（Everstream、Interos、Prewave）执行层弱，执行层强的（SAP Ariba、Kinaxis）情报层弱。** 没有人把"风险情报 → 影响量化 → 优化决策 → 闭环处置"真正打通。这个割裂就是我们的机会——做"情报到执行"的桥梁。

| **能力**     | **如果没有我们会怎样**                                       | **我们的差异化打法**                                         | **形成的壁垒**                                            |
| ------------ | ------------------------------------------------------------ | ------------------------------------------------------------ | --------------------------------------------------------- |
| 外部资讯处理 | 企业只能看到海量噪声资讯，无法快速筛出与自身供应链真正相关的事件。 | 用  Agent 做采集、去重、归类、摘要、实体映射和初评。         | 风险语料、标签体系、行业知识库与 prompt / workflow 资产。 |
| 风险影响评估 | 只能靠专家经验打分，难以复制和规模化。                       | 把关键性、替代性、金额、订单影响、时长、地理与节点脆弱性建成评分卡。 | 评分模型和可解释规则体系。                                |
| 风险传播推演 | 只能知道某供应商有事，却不知道哪些产品、订单和区域会受影响。 | 用图模型  + OR 传播逻辑做 network impact analysis。          | 供应链知识图谱与影响传播模型。                            |
| 处置建议     | 只能停留在“提醒风险”，不能真正帮助业务响应。                 | 把替代供应商、库存缓冲、调拨、加急、换料、重排产、合同动作做成优化模型。 | 算法模型库和行业参数库。                                  |
| 闭环运营     | 项目交付后价值递减。                                         | Agent 跟办 + 运营评价指标 + 案例沉淀 +  模型迭代。           | 运营数据和持续学习机制。                                  |

**三层业务规划**

- **第一层：产品业务**：建设标准化平台能力，形成可复制的软件产品和行业模块。

- **第二层：数据与情报业务**：围绕公开资讯、第三方数据、行业知识库、风险 taxonomy 建立可复用的数据资产。

- **第三层：模型与运营服务业务**：提供模型共创、托管运营、月度风险评审、策略优化等高附加值服务。



### 3.1 竞品观察：主流玩家都在做什么

国内市场目前还没有一个完全等价于 Everstream / Prewave / Interos 的本土纯血龙头；更常见的情况是不同厂商分别占住数据、流程、优化或平台中的一个位置。我们面对的不是“一个强敌”，而是“多类厂商分食价值”的组合竞争。

我将竞品分成三类：一类是"专项风险 intelligence 平台"，一类是"采购套件内嵌风险模块"，一类是"场景仿真/响应优化平台"。这三类合起来，基本覆盖了客户未来可能对比的能力边界。

- 共性一：外部风险监测早已不是简单爬虫，而是“事件采集 + 实体映射 + 风险评估 + workflow action”的组合。

- 共性二：多层供应链映射是核心门槛。没有 supplier-site-part-product-region 的关联网络，风险难以准确传播。

- 共性三：持续评分、分级预警和场景模拟是主流方向，说明客户后续一定会问到“如果发生怎么办”。

- 共性四：最新方向已经从 generative AI 走向 agentic AI 与嵌入式决策支持，说明市场正在从“看见”走向“自动协同处置”。

| **类型**                        | **代表玩家**                              | **普遍强项**                                                 | **对我们的威胁**                                             |
| ------------------------------- | ----------------------------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ |
| 企业数据/风控 intelligence 厂商 | 合合信息/启信慧眼、天眼查专业版、企查查等 | 供应商尽调、关系穿透、围串标/空壳识别、批量监控、插件/API 集成 | 最容易以小预算、低侵入方式切入；会先吃掉你们‘基础风控’价值   |
| 采购/SRM 平台厂商               | 用友、金蝶、商越、浪潮、甄云等            | 供应商管理、寻源招采、合同、协同、商城、流程与合规           | 最容易通过既有客情与集采预算把‘风险’做成模块，形成替代       |
| 控制塔/计划/优化厂商            | 悠桦林等                                  | 计划排程、库存优化、控制塔、执行可视化                       | 会从‘决策优化’层面对你们形成中长期压力，但外部 risk intelligence 普遍较弱 |
| 客户自建 + 总包集成             | 大型央企信息中心、行业 SI、集团级总包     | 最懂本企业流程，最符合内控与信创要求                         | 是国央企项目中的隐形头号对手；其本质是用内部资源吸收你的产品价值 |
| 海外模块/国际套件               | SAP Ariba Supplier Risk、Kinaxis 等       | 成熟的供应商风险与计划协同能力                               | 对全球化程度高的央企有吸引力，但本地化、部署与流程适配通常更难 |

### 3.4 AI Agent 的角色

AI Agent 的角色不是聊天机器人，而是**工作流中的数字同事**：

- 采集 Agent：资讯采集、去重、归类、摘要
- 实体映射 Agent：事件 → 供应商/物料/订单关联
- 研判 Agent：初评 + 可解释分析 + 知识问答
- 建议 Agent：调用 OR 模型生成处置方案
- 跟办 Agent：待办跟催、闭环审计、案例沉淀

关键原则：**外部风险用 AI + 专家协同判断，内部风险用规则引擎 + 自动触发。** 保留人在环上的审核与治理。

- 应该做成“工作流中的数字同事”：在资讯采集、事件初判、实体映射、案例检索、处置建议、待办跟催等节点承担明确任务。
- 应该保留“人在环”：特别是外部风险评估、重大风险升级、红色预警闭环等环节，必须保留专家或管理者审核。
- 不建议只做一个问答机器人：那样很难形成业务闭环，也无法体现我们的 OR 优化优势。
- 不建议让 Agent 直接全自动下发高风险处置：必须通过权限、审批、审计日志和可解释输出建立治理。

**运筹学能力应该产品化为“模型工厂”**：

- **风险传播模型：**沿 supplier-site-part-product-order-region 链路计算影响范围与时延。

1. 定义该行业的风险分类体系（风险类型、等级、标签目录）
2. 配置行业专属规则和阈值
3. 接入行业特定数据源
4. 沉淀行业 Playbook



## 九、明确不建议做的事

1. 不要一开始做"全行业、全风险、全数据"的超级平台。先围绕 1-2 个行业做纵深。
2. 不要把大部分资源投在大屏和炫酷可视化上。大屏是签单工具，不是产品核心。没有图谱、规则、流程和闭环，展示层价值很快衰减。
3. 不要把 Agent 当聊天入口。真正的效率提升来自"任务自动完成"，不是"多问几个问题"。
4. 不要只做预警不做处置建议。客户最终会问：所以我现在应该怎么办？
5. 不要承诺外部风险 100% 自动判断。外部风险需要 AI 与专家协同，保留human-in-the-loop。



# 十、**组织与合作建议**

## 内部组织：

| **角色**           | **建议配置**                   | **核心职责**                               | **为什么重要**                           |
| ------------------ | ------------------------------ | ------------------------------------------ | ---------------------------------------- |
| 产品负责人         | 1  名强 owner                  | 统一需求、路线图、版本节奏与商业化包装     | 这是平台型项目，不能只按项目经理思维推进 |
| 行业专家           | 采购/供应链/风控专家若干       | 定义风险分类体系、playbook、评分逻辑     | 保证产品真正贴业务                       |
| Agent/算法团队     | NLP/图谱/Agent编排             | 事件处理、实体链接、自动研判、问答与跟办   | 形成智能化生产力                         |
| OR 建模团队       | 运筹学/优化博士团队            | 影响传播、替代、库存、调拨、计划优化       | 形成高壁垒差异化                         |
| 数据与实施交付团队-FDE | 数据接入、主数据治理、集成实施 | 打通 ERP/SRM/采购/质检/订单/库存           | 决定项目成败的地基                       |
| 风险运营团队       | 分析师  + 客户成功             | 日报周报、重大事件处理、案例沉淀、模型迭代 | 把一次性交付变成长期收入                 |



## 外部组织合作生态

· 数据合作：天气/灾害（气象部门）、物流、企业工商与司法、价格与大宗商品、制裁/合规、行业研究。

· 实施合作：ERP/SRM/采购系统集成商。

· 行业合作：与通信设备/半导体/制造行业专家建立知识共创机制。

· 咨询合作：联合做供应链韧性诊断、风险运营体系设计。
